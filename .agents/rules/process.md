# Agent Process Rules

Rules for how the AI assistant should operate within this project.

## Implementation Notes
- **implementation-notes.md**: While implementing tasks, maintain a "implementation-notes.md" file in the project root.
- **Content**:
    - Decisions made that were not in the original spec.
    - Changes made to the existing codebase or planned approach during execution.
    - Tradeoffs considered and the rationale for the chosen path.
    - Any critical information, warnings, or "gotchas" for the user.
- **Lifecycle**: Create the file when starting a new task and update it as decisions are made. Remove or archive it once the task is fully completed and verified, or keep it if it contains long-term valuable documentation.

## Logging
- Always log work using the `l-log` command as specified in `common.md`.
- Include relevant tags, problem descriptions, and solutions in the log.
